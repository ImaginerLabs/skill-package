# 代码分析技术指南

## 分析方法论

### 1. 系统性代码探索

#### 项目结构分析
- **目录扫描**: 使用 `list_dir` 分析项目整体结构
- **文件类型识别**: 识别主要技术栈（前端、后端、全栈）
- **入口点定位**: 找到 main 文件、配置文件、启动脚本

#### 技术栈识别
```bash
# 常见技术栈检测模式
- package.json → JavaScript/Node.js 项目
- requirements.txt → Python 项目
- pom.xml → Java Maven 项目
- go.mod → Go 项目
- Cargo.toml → Rust 项目
```

### 2. 深度代码阅读

#### 代码理解策略
1. **自上而下**: 从入口文件开始，逐步深入
2. **自下而上**: 从具体实现反向推导架构
3. **问题驱动**: 围绕特定问题聚焦分析

#### 关键分析点
- 导入依赖关系（import/require）
- 函数调用链和数据处理流
- 状态管理和数据流动
- 错误处理和边界条件

### 3. 问题诊断技术

#### Bug 定位方法
1. **症状分析**: 根据错误信息定位相关代码
2. **数据流追踪**: 跟踪问题数据的流动路径
3. **状态检查**: 验证关键状态和条件判断
4. **边界测试**: 检查极端情况和边界条件

#### 性能问题分析
- 循环和递归优化
- 内存使用分析
- I/O 操作效率
- 算法复杂度评估

## 工具使用最佳实践

### codebase_search 使用策略
- 使用完整英文问题描述搜索意图
- 结合具体上下文限制搜索范围
- 使用多角度关键词组合搜索

### grep_search 精确匹配
- 转义特殊正则表达式字符
- 使用文件类型过滤提高精度
- 结合目录范围限制搜索结果

### 文件阅读策略
- 大文件使用分段阅读策略
- 优先阅读函数签名和类定义
- 关注注释和文档字符串

## 常见问题模式

### 前端项目常见问题
1. **React/Vue 组件问题**
   - 状态管理错误
   - 生命周期问题
   - 渲染性能问题

2. **JavaScript/TypeScript 问题**
   - 类型错误
   - 异步处理问题
   - 模块导入导出问题

### 后端项目常见问题
1. **API 接口问题**
   - 参数验证缺失
   - 错误响应格式
   - 性能瓶颈

2. **数据库操作问题**
   - SQL 注入风险
   - 事务处理问题
   - 连接池管理
