# Story 1.5.1: shadcn/ui 组件补全与统一

Status: review

## Story

As a 开发者,
I want 拥有完整的 shadcn/ui 组件基础设施,
So that 所有 UI 元素视觉风格统一，不再出现原生 HTML 元素与暗色主题的割裂。

## Acceptance Criteria

1. **AC1 — Input 组件**: `src/components/ui/input.tsx` 可用，支持 `className` 扩展，暗色主题 CSS Variables 映射，`:focus-visible` 焦点指示器（2px solid Run Green）
2. **AC2 — Select 组件**: `src/components/ui/select.tsx` 可用，导出 `Select`/`SelectTrigger`/`SelectValue`/`SelectContent`/`SelectItem`，基于 `@radix-ui/react-select`
3. **AC3 — Checkbox 组件**: `src/components/ui/checkbox.tsx` 可用，基于 `@radix-ui/react-checkbox`，暗色主题样式
4. **AC4 — Badge 组件**: `src/components/ui/badge.tsx` 可用，支持 `default`（绿色，分类）、`secondary`（蓝色，类型）、`outline`（灰色，标签）、`destructive`（红色）四种 variant
5. **AC5 — Separator 组件**: `src/components/ui/separator.tsx` 可用，基于 `@radix-ui/react-separator`，支持水平/垂直方向
6. **AC6 — Tooltip 组件**: `src/components/ui/tooltip.tsx` 可用，基于 `@radix-ui/react-tooltip`，导出 `Tooltip`/`TooltipTrigger`/`TooltipContent`/`TooltipProvider`
7. **AC7 — AlertDialog 组件**: `src/components/ui/alert-dialog.tsx` 可用，基于 `@radix-ui/react-alert-dialog`，导出完整子组件集
8. **AC8 — Dialog 组件**: `src/components/ui/dialog.tsx` 可用，基于 `@radix-ui/react-dialog`，导出完整子组件集
9. **AC9 — ScrollArea 组件**: `src/components/ui/scroll-area.tsx` 可用，基于 `@radix-ui/react-scroll-area`

11. **AC11 — 焦点指示器**: 所有组件支持 `:focus-visible` 焦点指示器
12. **AC12 — 类型检查**: `tsc --noEmit` 编译通过

## Dev Notes

### 现有组件模式（必须严格遵循）

**已有组件**: `button.tsx`、`card.tsx` — 位于 `src/components/ui/`

**Import 路径**: 组件内 import cn 使用 `import { cn } from "../lib/utils";`

- `cn()` 定义在 `src/lib/utils.ts`，使用 `clsx` + `tailwind-merge`

**组件模式**:

```tsx
import * as React from "react";
import { cn } from "../lib/utils";

const Component = React.forwardRef<HTMLElement, Props>(
  ({ className, ...props }, ref) => (
    <element ref={ref} className={cn("base-classes", className)} {...props} />
  ),
);
Component.displayName = "Component";
export { Component };
```

**CVA 模式**（有 variant 的组件如 Button、Badge）:

```tsx
import { cva, type VariantProps } from "class-variance-authority";
const variants = cva("base-classes", { variants: { ... } });
```

### 关键约束

- **禁止使用原生 `<select>`** — 必须使用 Radix Select 组件
- **禁止使用 `window.confirm()`** — 必须使用 AlertDialog 组件
- 所有组件 **必须** 使用 `React.forwardRef` 模式
- 所有组件 **必须** 设置 `displayName`
- 所有组件 **必须** 支持 `className` prop 扩展
- 文件命名: kebab-case（如 `alert-dialog.tsx`、`scroll-area.tsx`）
