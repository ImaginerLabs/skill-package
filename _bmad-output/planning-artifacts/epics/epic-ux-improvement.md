本 Epic 来源于对 Skill Manager 全页面的 UX 走查（Chrome DevTools 实测），共发现 **11 个用户体验问题**。

## Epic 1: 核心功能修复（P0）

**目标：** 修复阻断核心流程的功能性缺陷，确保搜索、工作流编排等核心功能可用。

---

### Story 1.1: 全局搜索实时过滤

As a 用户,
I want 在全局搜索弹窗中输入关键词后，结果列表能实时过滤，
So that 我可以快速找到目标 Skill 或页面，而不需要滚动浏览全量列表。

**⚠️ 误判检查（实现前必读）：**
> - 走查时观察到：输入"代码"后搜索结果未过滤，仍显示全量列表
> - **检查方式：** 打开 `http://localhost:5173/`，按 `Cmd+K` 打开搜索弹窗，输入任意关键词，观察列表是否变化
> - **可能的误判原因：** 搜索框可能未自动获得焦点，导致输入未被捕获；或搜索功能已实现但存在 debounce 延迟
> - **确认方式：** 搜索相关组件源码（如 `CommandPalette`、`SearchModal`），检查是否有 `onChange` / `onInput` 事件绑定和过滤逻辑

**验收标准：**

**Given** 用户打开全局搜索弹窗
**When** 用户在搜索框中输入关键词
**Then** 结果列表实时过滤，只显示名称/描述/标签包含关键词的条目
**And** 搜索框自动获得焦点，无需手动点击

**Given** 搜索结果为空
**When** 用户输入的关键词无匹配项
**Then** 显示"未找到相关结果"空状态提示

**Given** 用户在结果列表中
**When** 用户按上下方向键
**Then** 高亮项在结果间移动，按 Enter 跳转到对应页面/Skill

---

### Story 1.2: 工作流编排状态持久化

As a 用户,
I want 在工作流编排页面编辑的步骤在刷新后不丢失，
So that 我可以放心编辑，不担心意外刷新导致工作丢失。

**⚠️ 误判检查（实现前必读）：**
> - 走查时观察到：添加步骤后刷新页面，步骤全部消失
> - **检查方式：** 打开 `http://localhost:5173/workflow`，添加 2 个 Skill 步骤，刷新页面，观察步骤是否保留
> - **可能的误判原因：** 持久化功能可能已实现但存在 bug；或这是设计意图（新建工作流不持久化，已保存的工作流才持久化）
> - **确认方式：** 检查工作流相关 store/state 代码，确认是否有 localStorage 持久化逻辑；检查产品设计文档是否有说明

**验收标准：**

**Given** 用户在新建工作流页面添加了若干步骤
**When** 用户刷新页面
**Then** 之前添加的步骤自动恢复，工作流名称也保留

**Given** 用户在编辑工作流时准备离开页面
**When** 用户点击其他导航项
**Then** 弹出"是否保存草稿"确认提示，用户可选择保存或放弃

---

### Story 1.3: 工作流按钮禁用状态提示

As a 用户,
I want 当"预览"和"生成工作流"按钮不可点击时能看到原因说明，
So that 我知道还需要完成哪些操作才能继续。

**⚠️ 误判检查（实现前必读）：**
> - 走查时观察到：添加步骤后按钮仍为 disabled，无任何提示
> - **检查方式：** 打开工作流页面，添加步骤但不填写工作流名称，观察按钮状态；再填写名称，观察按钮是否激活
> - **可能的误判原因：** 按钮可能在填写名称后会自动激活，只是走查时未完成所有必填项；tooltip 可能已存在但走查时未触发
> - **确认方式：** 检查按钮的 `disabled` 条件逻辑，确认所有必填项是否有视觉标记

**验收标准：**

**Given** 用户未填写工作流名称
**When** 用户 hover 到"预览"或"生成工作流"按钮上
**Then** 显示 tooltip："请先填写工作流名称"

**Given** 工作流名称已填写但步骤列表为空
**When** 用户 hover 到"生成工作流"按钮上
**Then** 显示 tooltip："请至少添加一个 Skill 步骤"

**Given** 工作流名称已填写且至少有一个步骤
**When** 用户查看按钮状态
**Then** 按钮变为可点击状态

**Given** 工作流名称输入框
**When** 用户查看表单
**Then** 必填项有红色星号（*）标记

## Epic 2: 重要体验改善（P1）

**目标：** 修复影响核心体验的问题，提升新用户上手成功率。

---

### Story 2.1: 修复 Skill 分类数量统计

As a 用户,
I want 左侧分类面板显示的 Skill 数量与实际一致，
So that 我可以通过分类快速了解各类别下有多少 Skill，并通过点击分类过滤列表。

**⚠️ 误判检查（实现前必读）：**
> - 走查时观察到：所有分类数量显示为 0，点击分类无法过滤
> - **检查方式：** 打开 Skill 库页面，查看左侧分类面板的数字；点击某个分类，观察右侧列表是否变化
> - **可能的误判原因：** 分类标签的 key 可能与 Skill 的 tags 字段格式不匹配（如大小写、中英文差异）；或分类功能依赖后端数据，本地开发环境数据不完整
> - **确认方式：** 检查 Skill 数据的 `tags` 字段值，与分类配置的 key 对比，确认映射关系

**验收标准：**

**Given** Skill 库中有带标签的 Skill
**When** 用户查看左侧分类面板
**Then** 每个分类显示该分类下 Skill 的实际数量

**Given** 用户点击某个分类
**When** 分类被选中
**Then** 右侧 Skill 列表只显示该分类下的 Skill

---

### Story 2.2: 同步功能新手引导

As a 新用户,
I want 在未配置同步目标时有清晰的引导，
So that 我能快速完成同步配置，开始使用同步功能。

**⚠️ 误判检查（实现前必读）：**
> - 走查时观察到：顶部"未配置同步目标，点击配置"点击无响应；同步页面缺少路径格式说明
> - **检查方式：** 点击顶部提示文字，观察是否有跳转或弹窗；在同步页面点击"添加同步目标"，观察交互
> - **可能的误判原因：** 点击区域可能不是文字本身而是旁边的按钮；路径格式说明可能在 placeholder 中已有
> - **确认方式：** 检查顶部提示组件的点击事件绑定；检查同步页面输入框的 placeholder 内容

**验收标准：**

**Given** 用户未配置任何同步目标
**When** 用户点击顶部"未配置同步目标，点击配置"提示
**Then** 跳转到同步配置页面，或弹出配置引导弹窗

**Given** 用户在同步页面点击"添加同步目标"
**When** 输入框出现
**Then** placeholder 显示示例路径，如 `~/projects/my-project/.codebuddy/skills`

### Story 2.3: 导入操作反馈与错误处理

As a 用户,
I want 点击"扫描"按钮后看到明确的加载状态和结果反馈，
So that 我知道操作是否成功，以及失败时的原因。

**⚠️ 误判检查（实现前必读）：**
> - 走查时观察到：点击扫描后无 loading 状态，无成功/失败反馈
> - **检查方式：** 在导入页面点击"扫描"按钮，观察按钮和页面区域的状态变化
> - **可能的误判原因：** 扫描可能是同步操作，速度极快导致 loading 状态一闪而过；或本地环境路径存在，扫描成功但无文件，空状态未设计
> - **确认方式：** 检查扫描相关的异步逻辑，确认是否有 loading state 管理

**验收标准：**

**Given** 用户点击"扫描"按钮
**When** 扫描进行中
**Then** 按钮显示 loading 动画，扫描区域显示骨架屏或进度提示

**Given** 扫描完成但路径下无可导入文件
**When** 扫描结果为空
**Then** 显示"未找到可导入的 Skill 文件"空状态提示

**Given** 扫描路径不存在或无权限
**When** 扫描失败
**Then** 显示具体错误信息，如"路径不存在"或"权限不足，请检查目录权限"

### Story 2.4: 版本号与 package.json 同步

As a 用户/开发者,
I want 左下角显示的版本号与 package.json 中的 version 字段保持一致，
So that 我能准确知道当前运行的版本，便于问题排查和版本追踪。

**⚠️ 误判检查（实现前必读）：**
> - 走查时观察到：左下角版本号显示为静态硬编码值（如 `v0.1.0`），与 package.json 可能不同步
> - **检查方式：** 查看 `package.json` 中的 `version` 字段值，与页面左下角显示的版本号对比
> - **可能的误判原因：** 版本号可能已通过 Vite `define` 或 `import.meta.env` 注入，只是当前 package.json 版本恰好也是 `v0.1.0`，看起来像硬编码
> - **确认方式：** 搜索源码中版本号的引用位置（如 `v0.1.0` 或 `APP_VERSION`），确认是否为动态注入

**验收标准：**

**Given** 项目构建时
**When** Vite 构建执行
**Then** `package.json` 中的 `version` 字段被注入为环境变量（如 `import.meta.env.VITE_APP_VERSION`）

**Given** 用户查看页面左下角
**When** 页面加载完成
**Then** 显示的版本号与 `package.json` 中的 `version` 字段一致

**Given** 开发者修改 `package.json` 中的 `version` 字段
**When** 重新构建或热更新后
**Then** 页面版本号自动同步，无需手动修改任何组件代码

## Epic 3: 易用性优化（P2）

### Story 3.1: Skill 详情侧边栏快捷操作

As a 用户,
I want 在 Skill 详情侧边栏中有快捷操作按钮，
So that 我可以直接复制路径或同步到项目，而不需要切换到其他页面。

**⚠️ 误判检查（实现前必读）：**
> - 走查时观察到：详情侧边栏只有查看功能，无快捷操作
> - **检查方式：** 打开任意 Skill 的详情侧边栏，查看顶部操作区是否有"复制路径"等按钮
> - **可能的误判原因：** 快捷操作可能在右键菜单或更多操作（`...`）菜单中，走查时未发现
> - **确认方式：** 检查详情侧边栏组件，确认操作按钮的完整列表

**验收标准：**

**Given** 用户打开 Skill 详情侧边栏
**When** 用户查看顶部操作区
**Then** 显示"复制路径"按钮

**Given** 用户点击"复制路径"
**When** 点击操作完成
**Then** Skill 文件路径复制到剪贴板，显示"已复制"toast 提示

### Story 3.2: 工作流步骤描述说明优化

As a 用户,
I want 工作流步骤的描述输入框有清晰的说明，
So that 我知道填写描述的用途和格式要求。

**⚠️ 误判检查（实现前必读）：**
> - 走查时观察到：步骤描述输入框无 placeholder 说明
> - **检查方式：** 在工作流编排页面添加一个步骤，查看描述输入框的 placeholder 内容
> - **可能的误判原因：** placeholder 可能已存在，只是走查时截图分辨率不够清晰
> - **确认方式：** 检查步骤卡片组件的描述输入框 `placeholder` 属性值

**验收标准：**

**Given** 用户在工作流编排页面添加了一个步骤
**When** 用户查看步骤卡片中的描述输入框
**Then** placeholder 显示说明文字，如"描述此步骤的执行条件或注意事项（可选）"

### Story 3.3: 列表视图 Skill 详情入口

As a 用户,
I want 在列表视图中点击 Skill 行也能打开详情侧边栏，
So that 卡片视图和列表视图的交互行为保持一致。

**⚠️ 误判检查（实现前必读）：**
> - 走查时观察到：列表视图点击 Skill 行无响应，无法打开详情
> - **检查方式：** 切换到列表视图，点击任意 Skill 行，观察是否打开详情侧边栏
> - **可能的误判原因：** 列表视图可能需要点击特定区域（如名称文字）才能打开详情，而非整行点击
> - **确认方式：** 检查列表视图行组件的点击事件，确认是否绑定了打开详情的逻辑

**验收标准：**

**Given** 用户切换到列表视图
**When** 用户点击任意 Skill 行
**Then** 右侧打开该 Skill 的详情侧边栏，与卡片视图行为一致

### Story 3.4: 分类管理批量操作

As a 用户,
I want 在设置页面的分类管理中能批量将 Skill 移入/移出分类，
So that 我不需要逐个进入 Skill 详情页修改分类，提升管理效率。

**⚠️ 误判检查（实现前必读）：**
> - 走查时观察到：设置页面分类管理无法展开查看该分类下的 Skill，也无批量操作
> - **检查方式：** 打开设置页面，查看分类卡片是否可展开；检查是否有批量操作入口
> - **可能的误判原因：** 批量操作可能在 Skill 库页面通过多选实现，而非在设置页面；这可能是设计意图
> - **确认方式：** 检查设置页面分类组件，确认是否有展开/批量操作的设计；与产品设计稿对比

**验收标准：**

**Given** 用户在设置页面查看分类列表
**When** 用户点击某个分类卡片
**Then** 展开显示该分类下的 Skill 列表

**Given** 用户展开了某个分类
**When** 用户选择多个 Skill
**Then** 出现批量操作工具栏，支持"移出此分类"操作
