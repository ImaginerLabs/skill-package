## 一、执行摘要

Git 智能分支名生成器是一款面向开发团队的 Web 工具，解决 Git 分支命名不统一、规范难执行的核心痛点。产品差异化在于**中文描述智能转英文分支名**——开发者用中文输入需求描述，工具自动映射关键词并生成符合 Git 规范的分支名及对应命令，将命名耗时从 30 秒压缩至 5 秒内。

**核心价值主张：** 中文即英文，规范即默认。

**目标用户：** 后端/前端开发者、DevOps 工程师、技术负责人、开源贡献者。

**MVP 范围：** 单页 Web 应用，纯前端实现，无需后端服务，支持离线使用。

### FR-3：描述输入

| 属性 | 说明 |
|------|------|
| 功能描述 | 用户输入分支的简短描述，工具自动转换为英文短横线格式 |
| 优先级 | P0 |
| 验收标准 | 输入中文描述后，输出为全英文短横线分隔格式；输入英文描述后，直接格式化为短横线格式；不保留任何中文字符 |
| 痛点映射 | P-4 |

**核心能力 —— 中文关键词智能映射：**

工具内置 70+ 常见中文关键词到英文的映射表，自动将中文描述转为英文分支名片段。

**映射分类与示例：**

| 分类 | 示例映射 |
|------|----------|
| 用户与权限 | 用户→user, 登录→login, 注册→register, 权限→permission, 角色→role |
| 业务实体 | 订单→order, 支付→payment, 商品→product, 搜索→search, 购物车→cart |
| 技术操作 | 修复→fix, 优化→optimize, 重构→refactor, 部署→deploy, 缓存→cache |
| 系统运维 | 监控→monitor, 日志→log, 配置→config, 集群→cluster |
| 界面交互 | 页面→page, 弹窗→modal, 表单→form, 导航→nav |
| 异常处理 | 超时→timeout, 崩溃→crash, 异常→exception, 漏洞→vulnerability |

> 完整映射表详见实现文件中的 `chineseMap` 对象。映射表需覆盖以上 6 个分类，每个分类至少 10 个高频词条。

**格式化规则（按序执行）：**

1. 将中文关键词替换为对应英文
2. 移除所有剩余中文字符（未匹配的中文直接丢弃）
3. 下划线和空格转换为短横线 `-`
4. 移除所有非法字符（仅保留 `a-z`、`0-9`、`-`）
5. 合并连续短横线为单个
6. 去除首尾短横线
7. 全部转为小写

**示例：**

| 输入 | 处理过程 | 输出 |
|------|----------|------|
| 用户登录 | 用户→user, 登录→login | user-login |
| 修复支付超时问题 | 修复→fix, 支付→payment, 超时→timeout, "问题"无映射→丢弃 | fix-payment-timeout |
| 优化数据库查询 | 优化→optimize, "数据库"→database, "查询"→query | optimize-database-query |
| add user permission | 直通格式化 | add-user-permission |
| update_API_docs | `_` → `-`, 全小写 | update-api-docs |

### FR-4：关联单号（可选）

| 属性 | 说明 |
|------|------|
| 功能描述 | 添加 JIRA/TAPD 等任务单号，便于追溯 |
| 优先级 | P1 |
| 验收标准 | 单号自动转小写后拼接到分支名中；单号中的连字符保留，与分隔连字符用规则区分 |
| 痛点映射 | P-1 |

**输入示例：** `JIRA-1234`、`TAPD-5678`

**处理规则：**
- 自动转小写（`JIRA-1234` → `jira-1234`）
- 在分支名中，单号作为一个整体片段用短横线与其他部分连接（如 `jira-1234-payment-timeout`）

> **设计说明：** 单号内部的连字符（如 `jira-1234`）与片段间的分隔连字符视觉一致，但单号始终位于描述之前、日期之后，有固定位置可识别，不会引起歧义。

### FR-7：分支名生成

| 属性 | 说明 |
|------|------|
| 功能描述 | 根据用户输入的参数，按规范拼装生成分支名 |
| 优先级 | P0 |
| 验收标准 | 生成的分支名严格遵循拼接规则；空字段自动省略；生成的分支名必须通过 FR-8 校验 |
| 痛点映射 | P-1, P-3 |

**生成规则：**

```
<意图前缀>/[日期]-[单号]-[描述]-[作者]
```

**拼接规则：**
- 意图前缀与后续部分用 `/` 分隔（`/` 前无空格）
- `/` 后的可选部分（日期、单号、描述、作者）之间用 `-` 连接
- 仅拼接用户实际填写的部分，省略空字段
- 若无任何可选字段，仅保留 `<意图前缀>/<描述>`

**各字段位置固定（从前到后）：**

| 位置 | 字段 | 必填 |
|------|------|------|
| 1 | 意图前缀 + `/` | ✅ |
| 2 | 日期 | ❌ |
| 3 | 单号 | ❌ |
| 4 | 描述 | ✅ |
| 5 | 作者 | ❌ |

**生成示例：**

| 意图 | 描述 | 单号 | 日期 | 作者 | 生成分支名 |
|------|------|------|------|------|------------|
| feature | 用户登录 | — | — | — | `feature/user-login` |
| bugfix | 支付超时 | JIRA-123 | ✅ | — | `bugfix/20260417-jira-123-payment-timeout` |
| hotfix | 崩溃修复 | — | — | zhangsan | `hotfix/crash-fix-zhangsan` |
| release | v2.0发布 | — | — | — | `release/v2-0-publish` |
| refactor | 数据库查询优化 | TAPD-456 | ✅ | lisi | `refactor/20260417-tapd-456-optimize-database-query-lisi` |

> **注意：** 第 3 行示例中，"崩溃修复"经映射后为 `crash-fix`（崩溃→crash, 修复→fix），描述段为 `crash-fix`，与作者 `zhangsan` 用 `-` 连接，结果为 `hotfix/crash-fix-zhangsan`。

### FR-8：分支名校验

| 属性 | 说明 |
|------|------|
| 功能描述 | 对生成的分支名进行 Git 合法性校验 |
| 优先级 | P0 |
| 验收标准 | 所有校验规则执行后，返回通过/失败结果及具体错误信息；通过率目标 100%（SC-2） |
| 痛点映射 | P-3 |

**校验规则：**

| 规则 | 校验条件 | 错误提示 |
|------|----------|----------|
| 非空检查 | 分支名不能为空 | "分支名不能为空" |
| 长度限制 | 不超过 100 个字符 | "分支名过长，建议不超过100个字符" |
| 禁止连续点号 | 不能包含 `..` | "分支名不能包含连续的点号(..)" |
| 非法字符检查 | 不能包含空格、`~`、`^`、`:`、`?`、`*`、`[`、`\`、`@{` | "分支名包含非法字符" |
| 结尾检查 | 不能以 `/` 或 `.` 或 `-` 结尾 | "分支名不能以 /、. 或 - 结尾" |
| 开头检查 | 不能以 `-` 开头 | "分支名不能以 - 开头" |
| 锁定检查 | 不能包含 `@{` | "分支名不能包含锁定表示法(@{)" |
| 连续斜杠 | 不能包含 `//` | "分支名不能包含连续的斜杠(//)" |

## 十一、数据模型

### 11.1 核心数据结构

```typescript
// 分支生成输入
interface BranchInput {
  targetBranch: string;      // 目标分支
  intent: string;            // 意图 ID
  description: string;       // 原始描述
  ticketNumber?: string;     // 关联单号
  includeDate?: boolean;     // 是否包含日期
  author?: string;           // 作者标识
}

// 生成结果
interface BranchResult {
  branchName: string;        // 生成的分支名
  gitCommand: string;        // 生成的 Git 命令
  isValid: boolean;          // 是否通过校验
  errors: string[];          // 校验错误列表
}

// 历史记录条目
interface HistoryEntry {
  branchName: string;        // 分支名
  time: string;              // 生成时间（YYYY-MM-DD HH:mm）
}
```

| 键 | 值类型 | 说明 |
|----|--------|------|
